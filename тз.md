\## 1. Общие сведения



\*\*Полное наименование системы:\*\* Информационная система для автоматизации процесса формирования оптимальных карт раскроя листовых материалов.



\*\*Краткое наименование:\*\* ИС «Раскрой».



\*\*Тип системы:\*\* Локальное десктопное приложение.



\*\*Платформа:\*\* Windows 10/11, Linux (нативно или через Wine).



\*\*Язык интерфейса:\*\* Русский.



\*\*Область применения:\*\* мебельное производство, металлообработка, стекольная промышленность и иные отрасли, использующие листовые материалы.



\---



\## 2. Назначение и цели системы



\### 2.1 Назначение



Система предназначена для автоматизации задачи двумерного прямоугольного раскроя листовых материалов (2D Bin Packing / Cutting Stock Problem). Приложение обеспечивает формирование оптимальных карт раскроя, минимизирующих отходы материала, и управление связанными производственными данными.



\### 2.2 Цели



\- Повышение коэффициента использования материала (КИМ) до уровня не ниже \*\*85–90%\*\* для типовых заказов.

\- Снижение времени формирования карт раскроя с нескольких часов ручного труда до \*\*не более 60 секунд\*\* для заказа из 200 деталей.

\- Автоматизация учёта складских остатков и их списания.

\- Обеспечение наглядной визуализации результатов раскроя.



\### 2.3 Математическая постановка задачи



\*\*Дано:\*\*

\- Множество листов `S = {s₁, s₂, …, sₖ}`, где каждый лист `sⱼ` характеризуется шириной `Wⱼ`, высотой `Hⱼ` и типом материала.

\- Множество деталей `D = {d₁, d₂, …, dₙ}`, где каждая деталь `dᵢ` характеризуется шириной `wᵢ`, высотой `hᵢ`, количеством `qᵢ` и флагом вращения `rᵢ`.

\- Технологические параметры `T`: ширина реза `g`, отступ от края `b`, максимальная глубина гильотинного реза.



\*\*Найти:\*\* отображение `f : D → S × R²`, задающее для каждой детали номер листа и координаты `(x, y)` её левого нижнего угла, при котором:

\- детали не выходят за границы листов;

\- детали не перекрываются с учётом ширины реза `g`;

\- соблюдаются ограничения по вращению `rᵢ` и направлению волокон.



\*\*Целевая функция:\*\*



```

Z = Σ(Площадь деталей) / Σ(Площадь использованных листов) → max

```

\## 3. Характеристика объекта автоматизации



Объектом автоматизации является производственный процесс раскроя листовых материалов, включающий:



\- Ведение справочников материалов и форматов листов.

\- Приём и формирование заказов на изготовление деталей.

\- Расчёт оптимального плана раскроя.

\- Учёт и списание складских остатков.

\- Формирование выходной документации (карты раскроя, этикетки, ведомости).



\### Отраслевая специфика



| Отрасль | Ключевые требования |

|---|---|

| Мебельное производство (ЛДСП) | Учёт текстуры/волокон, ширина пропила 3–4 мм, кромление |

| Металлообработка | Минимальное расстояние между контурами, фигурный раскрой |

| Стекольная промышленность | Гильотинный рез, обход дефектов листа |

\## 4. Требования к системе



\### 4.1 Функциональные требования



\#### ФТ-01. Управление материалами

\- Ведение справочника типов материалов с атрибутами: название, описание.

\- Ведение справочника форматов листов: ширина, высота, толщина, тип материала.

\- Добавление, редактирование и удаление записей справочников.



\#### ФТ-02. Управление складом

\- Учёт физических листов на складе с указанием: формат, текстура, цена, количество, дефекты (в виде JSON-области).

\- Автоматическое списание листов после завершения задания раскроя.

\- Ведение истории движения склада (приход, списание, корректировка).

\- Отображение текущих остатков в разрезе форматов и типов материалов.



\#### ФТ-03. Работа с заказами

\- Создание, редактирование и удаление многопозиционных заказов.

\- Каждая позиция заказа содержит: название детали, ширина, высота, количество, допустимость вращения, направление волокон, приоритет.

\- Импорт спецификаций из форматов CSV и Excel (`.xlsx`).

\- Использование шаблонов заказов для быстрого создания типовых наборов деталей.

\- Группировка деталей по типу материала.



\#### ФТ-04. Оптимизация раскроя

\- Запуск алгоритма раскроя для выбранного заказа с указанием параметров:

&#x20; - тип раскроя: гильотинный / произвольный;

&#x20; - ширина реза (мм);

&#x20; - технологический отступ от края (мм);

&#x20; - максимальная глубина гильотинного реза;

&#x20; - алгоритм: жадная эвристика / генетический алгоритм / имитация отжига.

\- Возможность ручной корректировки результата: перемещение, вращение детали на листе.

\- Отображение прогресса вычислений.



\#### ФТ-05. Визуализация результатов

\- Графическое отображение карты раскроя в масштабируемом виде для каждого листа.

\- Цветовая маркировка деталей по позициям заказа.

\- Отображение отходов и технологических зазоров.

\- Навигация по листам задания.



\#### ФТ-06. Формирование выходной документации

\- Сводная ведомость: количество использованных листов, суммарная площадь деталей, площадь отходов, КИМ (%).

\- Экспорт карты раскроя в формат PDF и PNG.

\- Печать этикеток с наименованием детали и QR-кодом / штрихкодом.



\#### ФТ-07. Хранение истории расчётов

\- Сохранение результатов всех завершённых заданий раскроя в базе данных.

\- Просмотр и восстановление ранее выполненных расчётов.

\## 6. Архитектура системы



\### 6.1 Общая архитектура



Система реализуется как \*\*монолитное локальное приложение\*\* с чёткой модульной структурой. Взаимодействие компонентов строится по принципу:



```

Пользователь → GUI → Бизнес-логика → Алгоритм раскроя

&#x20;                                  ↕

&#x20;                             База данных (SQLite)

```



Пользовательский интерфейс \*\*не обращается к базе данных напрямую\*\* — все операции выполняются через слой бизнес-логики.



\### 6.2 Модульная структура



```

cutting\_app/

├── main.py                   # Точка входа

├── requirements.txt

├── config.py                 # Конфигурация приложения

│

├── db/                       # Модуль управления данными

│   ├── models.py             # SQLAlchemy-модели (ORM)

│   ├── session.py            # Управление сессией БД

│   └── migrations.py         # Инициализация и миграции схемы

│

├── core/                     # Бизнес-логика

│   ├── material\_service.py   # Сервис типов материалов и форматов

│   ├── stock\_service.py      # Сервис склада

│   └── export\_service.py     # Сервис экспорта (PDF, PNG, этикетки)
```

\## 8. Алгоритмическое ядро



\### 8.1 Общая схема работы



```

Входные данные (листы + детали + параметры)

&#x20;       ↓

Предобработка: сортировка деталей по убыванию площади

&#x20;       ↓

Жадная эвристика FFD → начальное решение

&#x20;       ↓

Метаэвристика (GA или SA) → улучшенное решение

&#x20;       ↓

Результат: координаты размещения + КИМ

```



\### 8.2 Жадная эвристика (First Fit Decreasing)



1\. Отсортировать детали по убыванию площади `(w × h)`.

2\. Для каждой детали найти первый лист и первую позицию (skyline-метод или corner-point метод), куда деталь помещается с учётом зазора `g`.

3\. Если ни один лист не подходит — открыть новый.

4\. Если допускается вращение (`rᵢ = true`) — проверить оба варианта ориентации.



\*\*Сложность:\*\* O(n·m), где n — число деталей, m — число листов.



\### 8.3 Генетический алгоритм (GA)



| Параметр | Значение по умолчанию |

|---|---|

| Размер популяции | 50 особей |

| Число поколений | 200 |

| Вероятность мутации | 0.05 |

| Вероятность скрещивания | 0.85 |

| Метод отбора | Турнирный (k=3) |



\- \*\*Хромосома:\*\* перестановка деталей (порядок подачи в жадную укладку).

\- \*\*Фитнес-функция:\*\* КИМ результирующей раскладки.

\- \*\*Скрещивание:\*\* PMX (частично-отображённое скрещивание).

\- \*\*Мутация:\*\* перестановка двух случайных позиций.



\### 8.4 Абстрактный интерфейс алгоритма



```python

class BaseCuttingAlgorithm(ABC):

&#x20;   @abstractmethod

&#x20;   def solve(

&#x20;       self,

&#x20;       sheets: list\[Sheet],

&#x20;       pieces: list\[Piece],

&#x20;       params: CuttingParams

&#x20;   ) -> CuttingResult:

&#x20;       ...

```



Все алгоритмы реализуют единый интерфейс, что позволяет легко добавлять новые методы.

\## 11. Требования к тестированию



\### 11.1 Unit-тесты



\- Тестирование жадного алгоритма на эталонных наборах деталей.

\- Тестирование генетического алгоритма: КИМ результата ≥ 80% на тестовых данных.

\- Тестирование сервисов: создание/удаление заказов, движение склада.

\- Тестирование импортёра: корректный разбор CSV и Excel.



\### 11.2 Интеграционные тесты



\- Полный цикл: создание заказа → запуск раскроя → сохранение результата → списание со склада.

\- Проверка ссылочной целостности базы данных.



\### 11.3 Нагрузочные тесты



\- Заказ из \*\*200 деталей\*\* должен рассчитываться не более \*\*60 секунд\*\* на ПК с CPU Intel Core i5 (или эквивалент).

\- Заказ из \*\*500 деталей\*\* — не более \*\*180 секунд\*\*.



\### 11.4 Инструменты



\- \*\*pytest\*\* — фреймворк тестирования.

\- \*\*pytest-cov\*\* — покрытие кода (целевой показатель: ≥ 70%).
